Previous work

Good Care Connection

  • Designed the app architecture and UI. The app has two main areas, one for hospitals and one for aftercare facilities. Aftercare facilities can register their information into the app, and hospitals can search for for facilities based on their requirements. They can then submit a placement request with an aftercare facility which the receive in the request through the app, via email or via phone text.
  • Technologies used - HTML/CSS (using Bootstrap framework and SASS), PHP (using Codeigniter framework), Javascript/Jquery for interactivity. I used Bootstrap to speed up the front end development time. Because the app is an enterprise app, it does did not need a lot of heavy custom design or graphics work and Bootstrap added most of the design elements I needed out of the box. Codeigniter also made sense to use for the built in functionality. Before I came onto the project, the app had no logical structure, and again, Codeigniter added a lot of functionality out of the box eliminating the need to us to spend a lot of time building our own infrastructure.
  • Features I built include:
    • A login system with varying levels of permissions for hospital users, aftercare users, hospital admin user, aftercare admin user, and an admin section for the application admin.
    • A search functions based on locations and mile radius. Hospitals can enter their zip code and set 10, 25, 50, 100 mile radius, as well as options for their patient’s medical needs.
    • Areas with special style sheets for printing. Once a user does a search for the information they need, they can print out facility information for their clients to view.
    • Admin areas. Facilities on both ends of the app have an admin user that is allowed to add/remove basic users from their facility and edit their facility information. The owners of the app have an admin area where they are able to edit the database tables from inside the app using an UI.
  • http://goodcareconnection.com
